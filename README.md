# Liverpool densification analysis — code and data documentation

Code accompanying the manuscript *[paper title — anonymised for review]* submitted to *[venue — anonymised for review]* by [Anonymous Authors].

This repository contains the R scripts used to (a) classify post-2013 residential addresses in the Liverpool City Region into densification types, (b) attach a set of independent variables on a 100 m grid, and (c) fit logistic regression models linking densification outcomes to neighbourhood characteristics.

> **For reviewers:** this is an anonymised copy. Author names, affiliations, and the manuscript title have been replaced with placeholders. The data folder (`Input_Data/`) is not included in the repository for size and licensing reasons; see [Input_Data/README.md](Input_Data/README.md) for the full list of sources.

## Repository layout

```
Liverpool_Paper/
├── Liverpool_Paper.Rproj         RStudio project file — open this first
├── 01_Dependent Variable.R       Classify addresses into densification types, aggregate to 100m grid
├── 02_Independent Variables.R    Attach explanatory variables to the grid
├── 03_Descriptive Analysis.R     Prevalence figures, boxplots, correlation matrix
├── 04_Statistical Analysis.R     Logistic regression models, regression-results table
├── 05_Descriptive_Stats_Regression.R   Descriptive-stats table for the regression sample
├── join_lucs_bldg.Rmd            Pre-processing: join LUCS conversion records to building footprints
├── exploratory/                  Exploratory notebooks (not part of the reported pipeline)
│   ├── 03_variable_inspection.Rmd
│   └── 03a_explanatory_maps.Rmd
├── Input_Data/                   Raw inputs (not tracked; see Input_Data/README.md)
├── R Output/                     Intermediate .gpkg outputs (not tracked, regenerated by scripts)
└── R Export/                     Final tables and figures used in the paper (not tracked)
```

## How to run

1. Open `Liverpool_Paper.Rproj` in RStudio. This sets the working directory to the project root, which all scripts assume.
2. Place the required input files under `Input_Data/` following the structure documented in [Input_Data/README.md](Input_Data/README.md). Several inputs are licensed (OS AddressBase, OS MasterMap, OS OpenMap Local) and cannot be redistributed here; the README lists the source and expected filename for each.
3. Run the scripts in numerical order. Each script writes its output to `R Output/` (intermediate geospatial files) or `R Export/` (tables and figures).

### Pipeline

| Step | Script | Produces |
|---|---|---|
| 0 (prep) | `join_lucs_bldg.Rmd` | `R Output/lucs_liv.gpkg` — LUCS conversion records joined to 2023 building footprints. Move to `Input_Data/LUCS/lucs_liv.gpkg` before running step 1, or rerun this whenever the LUCS or footprint data changes. |
| 1 | `01_Dependent Variable.R` | `R Output/classified_addresses.gpkg`, `R Output/grid_depvar.gpkg` |
| 2 | `02_Independent Variables.R` | `R Output/grid_full.gpkg` |
| 3 | `03_Descriptive Analysis.R` | Figures in `R Export/boxplots/`, `R Export/figures/`, plus correlation matrix |
| 4 | `04_Statistical Analysis.R` | `R Export/table_allgrid.docx`, `R Export/model_results_grouped.docx` |
| 5 | `05_Descriptive_Stats_Regression.R` | `R Export/table_descriptive_stats.docx`, `.csv` |

The two `.Rmd` files in `exploratory/` are diagnostic notebooks (variable distributions and choropleth maps). They are not part of the reported analysis but are kept for transparency.

## Software requirements

Developed under **R 4.x** on macOS and Windows. Required CRAN packages:

- **Spatial / data wrangling:** `sf`, `dplyr`, `tidyr`, `tidyverse`, `purrr`, `tibble`, `dbscan`, `nngeo`, `osmdata`, `vroom`, `classInt`, `forcats`, `readr`
- **Plotting:** `ggplot2`, `corrplot`, `corrgram`, `PerformanceAnalytics`, `gridExtra`, `scales`, `svglite`
- **Modelling / evaluation:** `pscl`, `marginaleffects`, `pROC`, `car`, `broom`
- **Tables / Word export:** `flextable`, `officer`, `knitr`

Install everything with:

```r
install.packages(c(
  "sf", "dplyr", "tidyr", "tidyverse", "purrr", "tibble", "dbscan", "nngeo",
  "osmdata", "vroom", "classInt", "forcats", "readr",
  "ggplot2", "corrplot", "corrgram", "PerformanceAnalytics", "gridExtra",
  "scales", "svglite",
  "pscl", "marginaleffects", "pROC", "car", "broom",
  "flextable", "officer", "knitr"
))
```

## Data availability

All input data sources are listed in [Input_Data/README.md](Input_Data/README.md), with provenance and expected file paths. Licensed datasets (OS AddressBase Premium, OS MasterMap, OS OpenMap Local) cannot be redistributed; the README documents the licensor and what to request. All other inputs are public:

- Built-up Areas 2011 — ONS
- Output Area Classification 2011 — ONS
- Index of Multiple Deprivation 2015, Income 2010 — DCLG
- Council Tax property statistics 2011 — HMRC
- Corine Land Cover 2012 — Copernicus
- Land Use Change Statistics — DCLG
- OpenStreetMap amenities and parks — fetched via the `osmdata` API at runtime in script 02
- Driving-time matrix to Liverpool/Manchester stations — precomputed via OpenRouteService

## Reproducibility notes

- Working directory is set automatically when the `.Rproj` file is opened in RStudio. Scripts no longer call `setwd()`; if you source them from outside RStudio, set the working directory to the repository root first.
- Script 02 currently downloads OSM amenities live via the `osmdata` API. An OSM snapshot is included as `Input_Data/liv_amenities_osm.gpkg` for reviewers who want byte-identical reproduction; see the note in [Input_Data/README.md](Input_Data/README.md) for how to switch.
- CRS used throughout: EPSG:27700 (British National Grid).
